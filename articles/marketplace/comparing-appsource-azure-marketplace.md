# <a name="comparing-appsource-and-azure-marketplace"></a>Comparación de AppSource y Azure Marketplace  
Los escaparates atienden requisitos únicos del cliente. Los escaparates permiten dirigirse a ellos según el rol. Puede ofrecer la solución o el servicio correctos en función del cliente.  

## <a name="understanding-the-differences-between-the-storefronts"></a>Diferencias entre los escaparates  
La elección de un escaparate empieza por identificar la audiencia de destino de su oferta. Si se dirige a ambas audiencias, puede publicar una vez para vender en ambos escaparates.  
 
Tenga en cuenta las ventajas adicionales de los escaparates.   

### <a name="appsource"></a>AppSource  

| Ventajas | Detalles | 
|:--- |:--- | 
| ***Flexibilidad de facturación*** | Aprovisiona una experiencia de evaluación. Actualmente no ofrece una opción de publicación habilitada para comercio. Se puede utilizar la infraestructura actual de pedidos y facturación sin necesidad de realizar inversiones o cambios adicionales. | 
| ***Facilitación de contacto con otros asociados*** | Se pueden vincular fabricantes de software independiente, integradores de sistemas y proveedores de servicios administrados a escenarios de implementación concretos. Es posible que realice vender de forma colaborativa a nuevos clientes. | 
| ***Automation*** | Se pueden vincular fabricantes de software independiente, integradores de sistemas y proveedores de servicios administrados a escenarios de implementación concretos. Es posible que realice vender de forma colaborativa a nuevos clientes. |  
| ***Varios tipos de nube*** | Actualmente no es compatible con Azure Stack, Azure Government o nubes regionales. | 
| ***Presentación en contexto a los clientes*** | Llegue a más clientes gracias a la experiencia en aplicación de productos de Microsoft como Dynamics 365, Power BI y Office 365. |  

### <a name="azure-marketplace"></a>Azure Marketplace  

| Ventajas | Detalles | 
|:--- |:--- | 
| ***Flexibilidad de facturación*** | En Virtual Machines, las opciones de facturación de pago por uso emplean modelos de venta directa web y contratos Enterprise (EA) de Microsoft. Las opciones de precios también incluyen una suscripción de nivel Gratis donde una oferta es gratis permanentemente. Las opciones de precios también incluyen una suscripción Pruébelo ahora. Una suscripción Pruébelo ahora es gratis en promoción durante un período limitado, transcurrido el cual la máquina virtual se convierte en una suscripción de pago. La activación de BYOL también es una opción de apoyo.<br />Tanto para las opciones de facturación como para las siguientes ofertas de VM, todos los recursos aprovisionados de Azure se facturan directamente al cliente.<ul> <li>Aplicaciones de Azure: aplicación administrada</li> <li>Aplicaciones de Azure: plantilla de soluciones</li> </ul> | 
| ***Facilitación de contacto con otros asociados*** | En este momento no se pueden vincular partners de entrega ni proveedores de servicio a la oferta. Esta funcionalidad se lanzará en 2018. | 
| ***Automation*** | En este momento no se pueden vincular partners de entrega ni proveedores de servicio a la oferta. Aproveche las ventajas de SaaS automatizado con aprovisionamiento de complementos. Utilice Aplicaciones de Azure: plantilla de soluciones para automatizar escenarios de implementación y recopilación de datos basados en SaaS. |  
| ***Varios tipos de nube*** | Publique en soluciones de nube pública y en el entorno local con los siguientes tipos.<ul> <li>Azure Stack</li> <li>Azure Government</li> <li>Nubes regionales, incluidas China y Alemania</li> </ul> | 
| ***Presentación en contexto a los clientes*** | Permita que su solución esté disponible en la experiencia en el portal de Azure para la búsqueda contextual con máquinas virtuales y Aplicaciones de Azure: plantilla de soluciones. |
