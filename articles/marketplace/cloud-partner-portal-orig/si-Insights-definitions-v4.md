<a name="seller-insights-definitions"></a>Definiciones de Seller Insights
=======================

En la siguiente tabla se incluyen definiciones de muchos de los términos usados en Seller Insights.

|  **Término**                                        |  **Definición**                                                                                                                              |
|  --------------------------------------------    |  ---------------------------------------------------------------------------------------------------------------------------------           |
| Azure License Type (Tipo de licencia de Azure)                               | Tipo de contrato de licencia que los clientes utilizan para comprar Azure. También conocido como Canal.                                                  |
| Azure License Type: Cloud Solution Provider (Tipo de licencia de Azure: Proveedor de soluciones en la nube)      | El cliente final adquiere Azure y su oferta de Marketplace a través de su proveedor de soluciones en la nube, que actúa como su revendedor.                 |
| Azure License Type: Enterprise (Tipo de licencia de Azure: Enterprise)                   | El cliente final adquiere Azure y su oferta de Marketplace a través de un contrato Enterprise, firmado directamente con Microsoft.                  |
| Azure License Type: Enterprise through Reseller (Tipo de licencia de Azure: Enterprise a través de revendedor)  | El cliente final adquiere Azure y su oferta de Marketplace a través de un revendedor que le facilita el contrato Enterprise con Microsoft.     |
| Azure License Type: Pay as You Go (Tipo de licencia de Azure: Pago por uso)                | El cliente final adquiere Azure y su oferta de Marketplace a través de un contrato de pago por uso, firmado directamente con Microsoft.                |
| Azure Subscription GUID (GUID de suscripción a Azure)                          | Identificador único global (GUID) de la suscripción a Azure que el cliente usó para comprar su oferta de Marketplace.                     |
| Charge Amount (CC) [Importe del cargo (CC)]                               | Cantidad que se cobra al cliente en la moneda de facturación del cliente (CC).                                                                 |
| Charge Amount (PC) [Importe del cargo (PC)]                               | Cantidad que se cobra al cliente en la *moneda de pago (PC)*.                                                                      |
| Charge Date (Fecha del cargo)                                      | Fecha en que se calculó el cargo del cliente, normalmente inmediatamente después del período de uso.                                             |
| Nombre de la instancia de nube                              | Microsoft Cloud donde se produjo la implementación de una VM. (Azure Goverment: instancia de Microsoft Cloud para el gobierno de EE. UU.; Azure China: instancia de Microsoft Cloud en China; Azure Alemania: instancia de Microsoft Cloud en Alemania; Azure Global: instancias de Microsoft Cloud para todas las demás ubicaciones globales)                                                          |
| Custom Meter Usage (Uso del medidor personalizado)                               | Unidades medidas que se han consumido en ofertas del medidor personalizado.                                                                                        |
| Cliente                                         | Cualquier Cliente o usuario final de Azure que adquiere, utiliza o ve una oferta publicada a través de Marketplace. Los clientes se pueden identificar por el *GUID de la suscripción a Azure* único.                                                                                                                                                                        |
| Ciudad del cliente                                    | Nombre de ciudad proporcionado por el cliente final.                                                                                                  |
| Customer Communication Language (Idioma de comunicación del cliente)                  | Idioma preferido por el cliente para la comunicación.                                                                                    |
| Customer Company Name (Nombre de la empresa del cliente)                            | Nombre de empresa proporcionado por el cliente final.                                                                                               |
| Customer Country (País del cliente)                                 | Nombre de país proporcionado por el cliente final.                                                                                               |
| Customer Currency (CC) [Moneda del cliente (CC)]                           | Moneda preferida del cliente para precios y facturación.                                                                              |
| Customer Email (Correo electrónico del cliente)                                   | Dirección de correo electrónico proporcionada por el cliente final.                                                                                              |
| Customer First Name (Nombre del cliente)                              | Nombre del cliente.                                                                                                              |
| Customer Last Name (Apellidos del cliente)                               | Apellidos del cliente.                                                                                                               |
| Customer Payment Type (Tipo de pago del cliente)                            | Tipo de instrumento de pago utilizado por el cliente.                                                                                         |
| Customer Postal Code (Código postal del cliente)                             | Código postal proporcionado por el cliente final.                                                                                                |
| Customer State (Estado del cliente)                                   | Estado proporcionado por el cliente final.                                                                                                      |
| Date Acquired (Fecha de adquisición)                                    | Primera fecha en que la suscripción a Azure adquirió cualquier oferta publicada por usted.                                                                  |
| Date Lost (Fecha de pérdida)                                        | Fecha en que la suscripción a Azure canceló todas las ofertas publicadas por usted.                                                                     |
| Estimated Extended Charge (CC) [Cargo total estimado (CC)]                   | Cargo total estimado de la cantidad de unidades de uso de una SKU determinada (en la moneda del cliente). Este valor puede no ser exacto debido a errores de truncamiento o redondeo.                                                                                                                                                                                           |
| Estimated Extended Charge (PC) [Cargo total estimado (PC)]                   | Cargo total estimado de la cantidad de unidades de uso de una SKU determinada en función de la conversión de moneda el día en que se calcula el uso (en la moneda del anunciante). Este valor puede no ser exacto debido a errores de truncamiento o redondeo.                                                                                                                |
| Estimated Payout (PC) [Pago estimado (PC)]                            | Pago estimado de la cantidad de unidades de uso de una SKU determinada en función de la conversión de moneda el día en que se calcula el uso (en la moneda del anunciante). Este valor puede no ser exacto debido a errores de truncamiento o redondeo.                                                                                                                |
| Estimated Price (PC) [Precio estimado (PC)]                             | Precio estimado de una unidad de uso de una SKU determinada en función de la conversión de moneda el día en que se calcula el uso (en la moneda del anunciante). Este valor puede no ser exacto debido a errores de truncamiento o redondeo.                                                                                                                                            |
| Final Collection Status (Estado de cobro final)                          | Para un cargo específico, estado más reciente del ciclo de vida de facturación y cobro. Puede ser: *Collection in Progress* (Cobro en curso), *Collected* (Cobrado), *Paid Out* (Pagado), *Refund* (Reembolso) o *Write Off* (Cancelación).                                                                                                                                                                                      |
| Final Collection Status: Collected (Estado de cobro final: Cobrado)               | El cargo se ha cobrado.                                                                                                                    |
| Final Collection Status: Collection In Progress (Estado de cobro final: Cobro en curso)  | Aún no se ha cobrado el cargo del cliente. Facturación de Microsoft Azure todavía está facturando o cobrando al cliente.             |
| Final Collection Status: Refund (Estado de cobro final: Reembolso)                  | El cargo se ha reembolsado (el cargo completo o una cantidad parcial).                                                                      |
| Final Collection Status: Write Off (Estado de cobro final: Cancelación)               | El cargo se ha cancelado como deuda incobrable.                                                                                                      |
| IsNew Customer (Es un cliente nuevo)                                   | El cliente implementó esta *SKU* por primera vez durante el mes natural.                                                               |
| Mercados                                           | Nombre de país proporcionado por el cliente final.                                                                                               |
| Marketplace License Type (Tipo de licencia de Marketplace)                         | Método de facturación de la oferta de Marketplace.                                                                                                 |
| Tipo de licencia de Marketplace: facturado a través de Azure   | Microsoft es su agente para esta oferta de Marketplace y factura a los clientes en su nombre. (Tarjeta de crédito de pago por uso o factura de Enterprise)       |
| Marketplace License Type: Bring Your Own License (Tipo de licencia de Marketplace: Traiga su propia licencia) | La VM requiere una clave de licencia proporcionada por el cliente para la implementación. Microsoft no factura a los clientes por su uso de esta oferta de Marketplace. |
| Marketplace License Type: Free (Tipo de licencia de Marketplace: Gratuita)                   | La oferta de Marketplace está configurada para ser gratis para todos los usuarios. Microsoft no factura a los clientes por su uso de esta oferta de Marketplace.    |
| Marketplace License Type: Microsoft as Reseller (Tipo de licencia de Marketplace: Microsoft como revendedor)  | Microsoft es su revendedor para esta oferta de Marketplace.                                                                                       |
| Marketplace Order (Pedido de Marketplace)                                | Para las máquinas virtuales, un pedido de Marketplace representa la implementación de una o muchas máquinas virtuales vinculadas a una sola *SKU* en una única suscripción a Azure. Un único pedido de Marketplace puede representar muchas implementaciones con tamaños de núcleo variables. Para las aplicaciones administradas y los servicios de desarrollo, un pedido de Marketplace representa la compra de una única *SKU* mediante un *GUID de la suscripción a Azure*.           |
| Microsoft Fee (CC) [Tarifa de Microsoft (CC)]                               | Tarifa de Microsoft para la transacción en la moneda del cliente.                                                                                 |
| Uso normalizado                                 | Horas de uso normalizadas para contabilizarse en el número de núcleos de VM implicados en el uso: [número de núcleos de VM] x [horas de uso sin procesar]. Las máquinas virtuales designadas como "SHAREDCORE" usan 1/6 (o 0,1666) como multiplicador de [número de núcleos de VM].                                                                                                                                       |
| Nombre de la oferta                                       | Nombre de la oferta de Marketplace.                                                                                                           |
| Tipo de oferta                                       | Tipo de solución.                                                                                                                        |
| Offer Type: Custom Meter Solution (Tipo de oferta: Solución de medidor personalizado)                | El cliente implementó la solución de medidor personalizado del Marketplace.                                                                     |
| Offer Type: Managed Application (Tipo de oferta: aplicación administrada)                  | El cliente compró una suscripción a la aplicación de servicio administrada.                                                                   |
| Offer Type: Single VM (Tipo de oferta: VM única)                            | El cliente implementó una imagen mediante la selección de una VM única.                                                                                     |
| Offer Type: Solution Template (Tipo de oferta: plantilla de solución)                    | El cliente implementó la plantilla de solución.                                                                                                |
| Order Cancel Date (Fecha de cancelación de pedido)                                | Fecha en que se canceló el pedido de Marketplace.                                                                                                 |
| Número de pedidos                                      | Número de pedidos (activos y cancelados) que este *GUID de la suscripción a Azure* específico tiene de cualquiera de sus ofertas.                               |
| Order Purchase Date (Fecha de compra del pedido)                              | Fecha en que se creó el *pedido de Marketplace*. Para las máquinas virtuales, se trata de la primera fecha en que la imagen se implementó en la suscripción a Azure del cliente. Las implementaciones posteriores de esa misma imagen en la misma suscripción a Azure se consideran un pedido.                                                                                                       |
| Order Status (Estado del pedido)                                     | Estado de un pedido de Marketplace en el momento en que se actualizaron por última vez los datos.                                                                   |
| Order Status: Active (Estado del pedido: Activo)                             | Para las aplicaciones administradas, el pedido está activo. Para las máquinas virtuales, el cliente tiene al menos una implementación de la imagen en su suscripción a Azure.   |
| Order Status: Canceled (Estado del pedido: Cancelado)                          | Para las aplicaciones administradas, el pedido está cancelado. Para las máquinas virtuales, el cliente ha eliminado todas las implementaciones de la *SKU* de su suscripción a Azure.        |
| OrderID                                          | Identificador único del pedido. Para las aplicaciones administradas, el cliente ha comprado una suscripción mensual al servicio. Para las máquinas virtuales, el cliente ha implementado la imagen. Las implementaciones posteriores de esa misma imagen en la misma suscripción a Azure se consideran un pedido.                                                                       |
| Payment Type: Card (Tipo de pago: tarjeta)                               | El cliente paga los cargos de Marketplace con una tarjeta de crédito.                                                                          |
| Payment Type: Invoice (Tipo de pago: factura)                            | El cliente paga los cargos de Marketplace con una factura.                                                                                 |
| Payout Amount (PC) [Importe de pago (PC)]                               | Importe que se le paga en su *moneda de pago (PC)* preferida.                                                                            |
| Payout Currency (PC) [Moneda de pago (PC)]                             | Moneda utilizada para sus pagos.                                                                                                          |
| Payout Date (Fecha de pago)                                      | Fecha en que se envió la solicitud de pago de Microsoft al banco.                                                                           |
| Payout Status (Estado del pago)                                    | Indica si la transacción está en el ciclo de vida de pago: *Paig Out* (Pagado), *Upcoming Payout* (Pago futuro) o *Not Ready for Payout* (No listo para el pago).                        |
| Payout Status: Not Ready for Payout (Estado de pago: No listo para el pago)              | La transacción no está lista para el pago. [Consulte *Final Collection Status* (Estado de cobro final) para obtener más detalles]                                                    |
| Payout Status: Paid Out (Estado de pago: Pagado)                          | La transacción se incluyó en un cálculo de pago anterior. Los valores positivos están pagados y los negativos se compensan en el importe total a pagar. |
| EPayout Status: Upcoming Payout (Estado de pago: Pago futuro)                   | La transacción está preparada para el pago y se incluirá en el siguiente cálculo de pago disponible.                                           |
| Preview SKU (SKU de vista previa)                                      | Ha etiquetado la *SKU* como de "vista previa" y solo las suscripciones a Azure que ha autorizado pueden implementar y usar esta imagen.                       |
| Price (CC) [Precio (CC)]                                       | Precio de una unidad de uso para una SKU determinada (en la moneda del cliente).                                                                  |
| Promotional Contact Opt In (Participación de contacto promocional)                       | Indica si el cliente ha optado de manera proactiva por el contacto promocional de los anunciantes. En este momento, no presentamos la opción a los clientes, por lo que hemos indicado "No" de forma general. Una vez implementada esta característica, comenzaremos a actualizarla en consecuencia.                                                                                          |
| Publisher Currency (PC) [Moneda del anunciante (PC)]                          | Moneda preferida del anunciante para el pago.                                                                                          |
| Raw Usage (Uso sin procesar)                                        | Horas de uso de la oferta de Marketplace.                                                                                                      |
| Reseller Email (Correo electrónico del revendedor)                                   | Dirección de correo electrónico del revendedor implicado en la venta al cliente final.                                                                  |
| Reseller Name (Nombre del revendedor)                                    | Nombre del revendedor de Microsoft que administra el cliente final.                                                                                |
| URI de recurso                                     | Identificador único para implementaciones de servicios de desarrollo o máquinas virtuales individuales.                                                                    |
| SKU                                              | Nombre de la *SKU* tal como se definió durante la publicación. Una oferta puede tener muchas *SKU*, pero una *SKU* solo puede asociarse a una única oferta.              |
| SKU Billing Type (Tipo de facturación de la SKU)                                 | Método de facturación de la *SKU*.                                                                                                             |
| SKU Billing Type: BYOL (Tipo de facturación de la SKU: BYOL)                           | La VM requiere una clave de licencia proporcionada por el cliente para la implementación. Microsoft no factura a los clientes por su uso de esta oferta de Marketplace.   |
| SKU Billing Type: Free (Tipo de facturación de la SKU: Gratuita)                           | La *SKU* está configurada para ser gratis para todos los usuarios. Microsoft no factura a los clientes por su uso de esta *SKU*.                            |
| SKU Billing Type: Microsoft as Reseller (Tipo de facturación de la SKU: Microsoft como revendedor)          | Microsoft es su revendedor para esta *SKU*.                                                                                                   |
| SKU Billing Type: Paid (Tipo de facturación de la SKU: De pago)                           | Microsoft es su agente para esta *SKU* y factura a los clientes en su nombre. (Tarjeta de crédito de pago por uso o factura de Enterprise)                   |
| SKU Billing Type: Trial (Tipo de facturación de la SKU: Prueba)                          | El cliente está en su período de prueba y se convertirá en cliente de pago si no cancela o elimina la suscripción.                                         |
| Tax Amount (CC) [Importe de impuestos (CC)]                                  | Cantidad de impuestos que se aplica al cliente en la *moneda del cliente (CC)*.                                                               |
| Fecha de transacción                                 | Fecha de la transacción registrada en los informes de pago.                                                                               |
| Tipo de transacción                                 | Tipo de transacción del que se está generando el informe.                                                                                              |
| Transaction Type: Charge (Tipo de transacción: Cobro)                         | La transacción es un valor positivo que representa la cantidad que se cobra al cliente.                                                         |
| Transaction Type: Customer Refund (Tipo de transacción: Reembolso de cliente)                | El cargo del cliente se ha reembolsado. Esta transacción es un valor negativo igual a la cantidad del cargo positivo del cliente. Puede identificar el valor positivo correspondiente, que anteriormente era “de pago” mediante la identificación de la transacción con los mismos valores de *Charge Date* (Fecha de cargo) y *Transaction Type* = "Charge" (Tipo de transacción = "Cargo") y *Final Collection Status* = "Refund" (Estado de cobro final = "Reembolso").                   |
| Transaction Type: Payout Adjustment (Tipo de transacción: ajuste de pago)              | La transacción representa un ajuste positivo o negativo aplicado a su saldo por Microsoft, que se crea para responder a un error de facturación o de pago anterior.                                                                                                                                                                                            |
| Transaction Type: Write off (Tipo de transacción: Cancelación)                      | El cargo del cliente se ha cancelado como deuda incobrable. Esta transacción es un valor negativo igual a la cantidad del cargo positivo del cliente. Puede identificar el valor positivo correspondiente, que anteriormente era “de pago” mediante la identificación de la transacción con los mismos valores de *Charge Date* (Fecha de cargo) y *Transaction Type* = "Charge" (Tipo de transacción = "Cargo") y *Final Collection Status* = "Write Off" (Estado de cobro final = "Cancelación"). |
| Transaction Type: Write off reversal (Tipo de transacción: revocación de cancelación)             | La transacción es un valor positivo que representa la revocación de una transacción cancelada anteriormente.                                         |
| Trial End Date (Fecha de fin de la prueba)                                   | La fecha del período de prueba de este pedido va a finalizar o ha finalizado.                                                                              |
| Uso                                            | Uso notificado de la *SKU* por parte del cliente. Para las imágenes de máquina virtual, los registros de uso representan el uso durante el período notificado del tamaño de la máquina virtual y la *SKU*.   |
| Usage End Date (Fecha de finalización del uso)                                   | Fecha de finalización del período de uso del que se está generando el informe.                                                                                             |
| Fecha de uso                                       | Fecha en que se produjo el uso por parte del cliente.                                                                                                             |
| Usage Reference (Referencia de uso)                                  | El identificador de uno o varios días de uso del cliente para una SKU determinada asociada con una entrada del informe de pagos.                         |
| Usage Start Date (Fecha de inicio del uso)                                 | Fecha de inicio del período de uso del que se está generando el informe.                                                                                           |
| Usage Type (Tipo de uso)                                       | Descripción del uso que se está midiendo. [*Normalized Usage* (Uso normalizado) o *Raw Usage* (Uso sin procesar)]                                                               |
| Usage Units (Unidades de uso)                                      | Unidad de medida del uso indicado. Las máquinas virtuales siempre se miden con unidades de media horarias.                                      |
| Tamaño de VM                                          | Representa el tamaño del hardware de máquina virtual alineado con la oferta de Azure. Algunos ejemplos son `Basic_A0`, `Standard_A11`, `Standard_D12` y `Standard_G4`.   |
|  |  |
