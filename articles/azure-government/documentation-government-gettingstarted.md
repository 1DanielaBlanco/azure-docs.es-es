#  Introducción a Azure Government

##  Instrucciones generales para clientes de Azure Government

Dado que la mayoría del contenido técnico que está disponible actualmente da por supuesto que las aplicaciones se están desarrollando para el servicio Global en lugar de Microsoft Azure Government, es importante asegurarse de que los desarrolladores conozcan las diferencias principales para las aplicaciones desarrolladas para hospedarse en Azure Government.

- En primer lugar, hay servicios y características diferentes. Esto significa que ciertas características que se encuentran en regiones específicas del servicio Global pueden no estar disponibles en Azure Government.

- En segundo lugar, para las características que se ofrecen en Azure Government, hay diferencias de configuración desde el servicio Global. Por lo tanto, debe revisar el código de ejemplo, las configuraciones y los pasos asegurarse de que genera y ejecuta dentro del entorno de servicios en la nube de Azure Government.

- En tercer lugar, debe consultar la documentación de los servicios técnicos de Azure Government disponible en este sitio para ver la información que identifica el límite de Azure Government y las instrucciones y los procedimientos recomendados para los datos regulados o controlados del cliente.

Para información complementaria y actualizaciones, suscríbase al <a href="https://blogs.msdn.microsoft.com/azuregov/">blog de Microsoft Azure Government. </a>.

##  ITAR de Azure Government

La información de este sitio va dirigida a los clientes de Azure Government con obligaciones bajo la ITAR (International Traffic in Arms Regulations, Normativa de tráfico de armas internacional) que pretenden usar los servicios de Azure Government para almacenar, procesar o transmitir información regulada. La información proporcionada describe las funcionalidades de los servicios de Microsoft Azure Government y ofrece instrucciones generales aplicables a clientes de ITAR. Antes de incluir datos controlados por ITAR en su suscripción de Azure Government, debe familiarizarse con las funcionalidades de Azure Government y consultar a su equipo de cuentas en caso de duda.

Para ver información actual sobre los servicios de Azure Government que se incluyen en ITAR, consulte la <a href="http://www.microsoft.com/es-ES/TrustCenter/Compliance/default.aspx/"> página de cumplimiento del Centro de confianza de Microsoft Azure</a>. Puede que también haya otros servicios de Microsoft disponibles, pero no se encuentran dentro del ámbito de los servicios cubiertos por Azure Government. Los servicios de Azure Government podrían también permitirle usar diversos recursos, aplicaciones o servicios adicionales que proporcionan terceros o Microsoft bajo términos de uso y directivas de privacidad independientes. El usuario es responsable de la revisión de los términos de tales ofertas "complementarias", como las ofertas de Marketplace, a fin de asegurarse de que satisfacen sus necesidades con respecto al cumplimiento de ITAR.

<a href="https://azure.microsoft.com/es-ES/features/gov/"> Azure Government </a> está disponible para entidades que administran datos que están sujetos a determinados requisitos y regulaciones gubernamentales, como ITAR, donde se requiere el uso de Azure Government para cumplirlos. Los clientes de Azure Government están sujetos a una validación de idoneidad. La validación de idoneidad de Microsoft incluye la confirmación de que el usuario es fabricante, exportador o agente de artículos de defensa de acuerdo con el ITAR, según indica su registro en el Departamento de estado de EE. UU. o mediante un acuerdo de patrocinio con una entidad pública que tenga necesidades específicas para la administración de datos.

Las entidades que tengan dudas sobre su idoneidad para Azure Government deben consultar a su equipo de cuentas.
