# <a name="azure-government-compliance"></a>Cumplimiento de Azure Government
## <a name="blueprint-customer-responsibilities-matrix-streamlining-ato-processes"></a>Matriz de responsabilidades del cliente de Blueprint: simplificación de los procesos ATO
El programa Azure Blueprint permite a los clientes tener acceso a documentación para ayudar en los procesos de funcionamiento autorizado (ATO) específicos del sistema o la agencia. En este artículo se ahonda en la Matriz de responsabilidades del cliente (CRM) de Azure Blueprint y en cómo puede aprovechar este documento el personal de seguridad.

La CRM de Azure Blueprint se ha diseñado para la utilidad. Es un formato de hoja de cálculo que define SOLO los requisitos de control allí donde la compilación del cliente en Azure debe implementar el control de seguridad.

> [!NOTE]
> Dentro del contexto de Azure Blueprint, el "cliente" hace referencia a la compilación de la organización directamente en el entorno de IaaS/PaaS de Azure. Los clientes de Azure podrían incluir la compilación de ISVs de terceros en nombre de la compilación del gobierno o los organismos gubernamentales directamente en Azure.
> 
> 

La CRM proporciona los controles <a href="http://nvlpubs.nist.gov/nistpubs/SpecialPublications/NIST.SP.800-53r4.pdf"> NIST SP 800-53 Revision 4 </a> explícitos que incluyen la responsabilidad de un cliente. Esto incluye controles que son en su totalidad responsabilidad del cliente, así como controles que tienen una responsabilidad compartida entre el cliente y Azure. Los controles se desglosan a la granularidad de subrequisitos de control para ser tan explícitos como sea posible.

Por ejemplo, AC-1 requiere la documentación de procedimientos y directivas de control de acceso para el sistema que busca ATO. Para este control, Azure cuenta con procedimientos y directivas internos relacionados con mecanismos de control de acceso usados para administrar la infraestructura de Azure y la plataforma Azure. Los clientes también deben crear sus propios procedimientos y directivas de Access Control usados en su sistema específico creado en Azure. Los documentos de la CRM controlan las partes AC-1a, que requieren que los procedimientos y directivas incluyan contenido específico, así como AC-1b, que requiere que los clientes revisen y actualicen estos documentos al menos anualmente. 

Para tener acceso a la CRM, así como a la plantilla de Plan de seguridad de sistema de Azure Blueprint, envíe un correo electrónico a AzureBlueprint@microsoft.com.

Apreciamos cualquier comentario sobre la CRM, ya que es fundamental para que nuestra iteración del documento sea tan eficaz como sea posible en cuanto a la simplificación de procesos ATO del cliente.
