# <a name="about-the-virtual-machine-agent-and-extensions-for-windows-vms"></a>Acerca de las extensiones y del agente de máquina virtual para máquinas virtuales Windows

> [!IMPORTANT]
> Azure tiene dos modelos de implementación diferentes para crear recursos y trabajar con ellos: [Resource Manager y el clásico](../../../resource-manager-deployment-model.md). En este artículo se trata el modelo de implementación clásico. Microsoft recomienda que las implementaciones más recientes usen el modelo de Resource Manager. Para más información sobre los agentes y las extensiones de máquina virtual mediante Resource Manager, consulte [aquí](../extensions-features.md?toc=%2fazure%2fvirtual-machines%2fwindows%2ftoc.json).
