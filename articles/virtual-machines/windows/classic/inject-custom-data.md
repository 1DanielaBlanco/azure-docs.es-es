# <a name="injecting-custom-data-into-an-azure-virtual-machine"></a>Inyección de datos personalizados en una máquina virtual de Azure
> [!IMPORTANT] 
> Azure tiene dos modelos de implementación diferentes para crear recursos y trabajar con ellos: [Resource Manager y el clásico](../../../resource-manager-deployment-model.md). En este artículo se trata el modelo de implementación clásico. Microsoft recomienda que las implementaciones más recientes usen el modelo del Administrador de recursos. Para más información sobre el uso de la extensión de script personalizado con el modelo de Resource Manager, consulte [aquí](../extensions-customscript.md?toc=%2fazure%2fvirtual-machines%2fwindows%2ftoc.json).
