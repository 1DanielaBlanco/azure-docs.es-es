El clúster constará de un nodo principal local (un equipo con el sistema operativo Windows Server y HPC Pack) y otros nodos de ejecución que implemente localmente como instancias de rol de trabajo en un servicio en la nube de Azure. A continuación, podrá ejecutar trabajos informáticos en el clúster híbrido.\</p\> \<p\>\<img src="./media/set-up-hybrid-cluster-microsoft-hpc-pack-2012-sp1/hybrid\_cluster\_overview.png" alt="Hybrid HPC cluster" /\>\</p\> \<p\>Este tutorial muestra un enfoque, en ocasiones denominado clúster "ráfaga en la nube", para usar recursos informáticos escalables y bajo demanda en Azure a fin de ejecutar aplicaciones informáticas que consumen numerosos recursos.\</p\> \<p\>En este tutorial se supone que no cuenta con experiencia previa con los clústeres informáticos o con HPC Pack. Únicamente está destinado a ayudarle a implementar un clúster de proceso híbrido rápidamente con fines de demostración. Para obtener más información, así como pasos para implementar un clúster híbrido HPC Pack a mayor escala en un entorno de producción, consulte la \<a href="http://go.microsoft.com/fwlink/p/?LinkID=200493"\>información detallada\</a\>.\</p\> \<blockquote\> \<p\>[WACOM.NOTE] Azure ofrece una \<a href="http://go.microsoft.com/fwlink/p/?LinkId=389844"\>serie de tamaños\</a\> para los recursos informáticos, adecuados para las diferentes cargas de trabajo. Por ejemplo, las instancias A8 y A9 combinan alto rendimiento y acceso a baja latencia, aunque se necesita una red de aplicaciones de alto rendimiento para determinadas aplicaciones de HPC. Por ejemplo, consulte \<a href="http://go.microsoft.com/fwlink/p/?Linkid=328042"\>Sobre las instancias informáticas intensivas A8 y A9\</a\>.\</p\> \</blockquote\> \<p\>Este tutorial le guiará a través de estos pasos básicos:\</p\> \<ul\> \<li\>\<a href="\#BKMK\_Prereq"\>Requisitos previos\</a\>\</li\> \<li\>\<a href="\#BKMK\_DeployHN"\>Instalación de HPC Pack en el nodo principal\</a\>\</li\> \<li\>\<a href="\#BKMK\_Prpare"\>Preparación de la suscripción de Azure\</a\>\</li\> \<li\>\<a href="\#BKMK\_ConfigHN"\>Configuración del nodo principal\</a\>\</li\> \<li\>\<a href="\#BKMK\_worker"\>Incorporación de nodos de Azure al clúster\</a\>\</li\> \<li\>\<a href="\#BKMK\_start"\>Inicio de los nodos de Azure\</a\>\</li\> \<li\>\<a href="\#BKMK\_RunCommand"\> Ejecución de un comando en el clúster\</a\>\</li\> \<li\>\<a href="\#BKMK\_RunJob"\>Ejecución de un trabajo de prueba\</a\>\</li\> \<li\>\<a href="\#BKMK\_stop"\> Detención de los nodos de Azure\</a\>\</li\> \</ul\> \<h2 id="BKMK\_Prereq"\>Requisitos previos\</h2\> \<blockquote\> \<p\>[WACOM.NOTE] Necesita una cuenta de Azure para completar este tutorial. En caso de no tener ninguna, puede crear una cuenta de evaluación gratuita en tan solo unos minutos. Para obtener más información, consulte \<a href="http://www.windowsazure.com/es-es/develop/php/tutorials/create-a-windows-azure-account/"\>Creación de una cuenta de Azure\</a\>.\</p\> \</blockquote\> \<p\>Además, necesitará satisfacer los siguientes requisitos para este tutorial:\</p\> \<ul\> \<li\>\<p\>Un equipo local que ejecute una edición de Windows Server 2012 R2 o Windows Server 2012. Por ejemplo, use \<strong\>clusrun\</strong\> para obtener la configuración IP de los nodos de Azure.\</p\> \<ol\> \<li\>\<p\>En el nodo principal, abra un símbolo del sistema.\</p\>\</li\> \<li\>\<p\>Escriba el siguiente comando:\</p\> \<p\>\<code data-inline="1"\>clusrun /nodes:azurecn\* ipconfig\</code\>\</p\>\</li\> \<li\>\<p\>Verá un resultado similar al siguiente.\</p\> \<p\>\<img src="./media/set-up-hybrid-cluster-microsoft-hpc-pack-2012-sp1/clusrun1.png" alt="Clusrun" /\>\</p\>\</li\> \</ol\> \<h2 id="BKMK\_RunJob"\>Ejecución de un trabajo de prueba\</h2\> \<p\>Puede enviar un trabajo de prueba que se ejecute en el clúster híbrido. Este ejemplo es un sencillo trabajo de "barrido paramétrico" (un tipo de computación intrínsecamente paralelo) que ejecuta subtareas que se agregan un entero a sí mismas usando el comando \<strong\>set /a\</strong\>. Deje los valores predeterminados para el resto de la configuración y, a continuación, haga clic en \<strong\>Submit\</strong\> para enviar el trabajo.\</p\> \<p\>\<img src="./media/set-up-hybrid-cluster-microsoft-hpc-pack-2012-sp1/param\_sweep1.png" alt="Parametric Sweep" /\>\</p\>\</li\> \<li\>\<p\>Cuando finalice el trabajo, haga doble clic en el trabajo \<strong\>My Sweep Task\</strong\>.\</p\>\</li\> \<li\>\<p\>Haga clic en \<strong\>View Tasks\</strong\> y, a continuación, haga clic en una subtarea para ver la salida calculada de dicha subtarea.\</p\> \<p\>\<img src="./media/set-up-hybrid-cluster-microsoft-hpc-pack-2012-sp1/view\_job361.png" alt="Task Results" /\>\</p\>\</li\> \<li\>\<p\>Para ver qué nodo ha realizado el cálculo en esa subtarea, haga clic en \<strong\>Allocated Nodes\</strong\>.
