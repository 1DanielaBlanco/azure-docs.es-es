<h1 id="how-to-use-media-services">Uso de Servicios multimedia </h1> 
<p>En esta guía se muestra cómo empezar a programar con Servicios multimedia de Azure. La guía consta de un conjunto de artículos e incluye una introducción técnica a Servicios multimedia, pasos para configurar la cuenta de Azure para Servicios multimedia, una guía de configuración para el desarrollo y temas que muestran cómo llevar a cabo tareas de programación típicas. Entre los escenarios demostrados se encuentran los siguientes: carga, cifrado, codificación y entrega de recursos. Los ejemplos se escriben en C# y usan el SDK de Servicios multimedia para .NET. Para obtener más información sobre Servicios multimedia de Azure, consulte el tema  <a href=" #next-steps">Siguientes pasos </a>. </p>  <p>También puede programar Servicios multimedia con las API de REST basadas en OData. Puede generar una aplicación que haga llamadas de la API de REST a Servicios multimedia desde lenguajes .NET u otros lenguajes de programación. Para obtener una serie de documentación completa sobre la programación con la API de REST de Servicios multimedia, consulte  <a href="http://go.microsoft.com/fwlink/?linkid=252967">Entrega de contenido </a>. </p>  <p>Para comenzar a programar con la API de REST de Servicios multimedia o el SDK de Servicios multimedia, primero habilite su cuenta de Azure para Servicios multimedia como se describe en la sección  <a href=" #setup-account">Configuración de una cuenta de Azure para Servicios multimedia </a>. </p>  <p>La documentación del SDK de Servicios multimedia más actualizada se encuentra  <a href="http://msdn.microsoft.com/es-es/library/hh973613.aspx">aquí </a>. </p>  <h2 id="what-are-media-services"> <a name="what-are"> </a> <span class="short header">¿Qué son los Servicios multimedia? </span> </h2>  <p>Los Servicios multimedia de Azure forman una plataforma multimedia extensible que integra lo mejor de la plataforma multimedia de Microsoft y los componentes multimedia de terceros en Azure. Los Servicios multimedia proporcionan un proceso en la nube que permite a los socios del sector ampliar o reemplazar las tecnologías de componentes. Los vendedores independientes de software y los proveedores multimedia pueden usar Servicios multimedia para generar soluciones multimedia de un extremo a otro. Esta introducción describe la arquitectura general y los escenarios de desarrollo comunes para Servicios multimedia. </p>  <p>El siguiente diagrama muestra la arquitectura básica de Servicios multimedia. </p>  <p> <img src="./media/media-services-dotnet-how-to-use/wams-01.png" alt="Media Services Architecture" /> </p>  <h3 id="media-services-feature-support">Compatibilidad de las características de Servicios multimedia </h3>  <p>La versión actual de Servicios multimedia ofrece el siguiente conjunto de características para el desarrollo de aplicaciones multimedia en la nube. </p>  <ul>  <li> <strong>Introducción </strong>. Las operaciones de introducción introducen los recursos en el sistema, por ejemplo mediante su carga y cifrado antes de que se coloquen en el servicio de almacenamiento de Azure. Servicios multimedia ofrece integración con los componentes de socios para proporcionar soluciones rápidas de carga de UDP (protocolo de datagramas de usuario). </li>  <li> <strong>Codificación </strong>. Las operaciones de codificación incluyen la codificación, transformación y conversión de recursos multimedia. Puede ejecutar tareas de codificación en la nube con el codificador multimedia de Azure. Las opciones de codificación incluyen las siguientes: </li>  <li>Use el codificador multimedia de Azure y trabaje con una amplia variedad de códecs y formatos estándar, incluidos la Transmisión por secuencias suave IIS líder del sector, MP4 y la conversión a Transmisión por secuencias en directo HTTP de Apple. </li>  <li>Convierta bibliotecas enteras o archivos individuales con un control total sobre la entrada y la salida. </li>  <li>Un conjunto grande de tipos de archivo, formatos y códecs compatibles (vea  <a href="http://msdn.microsoft.com/es-es/library/dn535852.aspx">Tipos de archivo compatibles para Servicios multimedia </a>). </li>  <li>Conversiones de formato admitidas. Los Servicios multimedia le permiten convertir ISO MP4 (.mp4) a Formato de archivo de transmisión por secuencias suave (PIFF 1.3) (.ismv; .isma). También puede convertir Formato de archivo de transmisión por secuencias suave (PIFF) a Transmisión por secuencias en directo HTTP de Apple (.msu8, .ts). </li>  <li> <strong>Protección </strong>. La protección de contenido implica el cifrado de la transmisión en directo o contenido bajo demanda de cara a la seguridad en la entrega, el almacenamiento y el transporte. Los Servicios multimedia ofrecen una solución independiente de la tecnología DRM para proteger el contenido. Las tecnologías DRM compatibles actualmente son Microsoft PlayReady y cifrado común de MPEG. También habrá compatibilidad con tecnologías DRM adicionales. </li>  <li> <strong>Transmisión por secuencias </strong>. La transmisión de contenido implica el envío de este en directo o bajo demanda a los clientes, o puede descargar archivos multimedia específicos desde la nube. Los Servicios multimedia ofrecen una solución independiente del formato para el contenido por secuencias. Los Servicios multimedia proporcionan compatibilidad con el origen de la transmisión en formatos MP4, Transmisión por secuencias en directo HTTP de Apple y Transmisión por secuencias suave. Se ofrecerá compatibilidad con formatos adicionales. También puede entregar directamente contenido por secuencias usando un CDN de terceros, que permite la opción de realizar una escalación a millones de usuarios. </li>  </ul>  <h3 id="media-services-development-scenarios"> Escenarios de desarrollo de Servicios multimedia </h3> <p>Los Servicios multimedia son compatibles con escenarios de desarrollo multimedia comunes, como se describe en la siguiente tabla.<br/><table border="2" cellspacing="0" cellpadding="5" style="border: 2px solid  #000000;"> <br />  <thead> <br />  <tr> <br />  <th>Escenario </th> <br />  <th>Descripción </th> <br />  </tr> <br />  </thead> <br />  <tbody> <br />  <tr> <br />  <td>Creación de flujos de trabajo de un extremo a otro </td> <br />  <td>Genere flujos de trabajo multimedia completos en toda la nube. Desde la carga de medios a la distribución de contenido, los Servicios multimedia ofrecen una serie de componentes que pueden combinarse para administrar flujos de trabajo específicos de aplicaciones. Entre las capacidades actuales se encuentran la carga, el almacenamiento, el cifrado, la conversión de formato, la protección de contenido y la entrega de transmisión por secuencias bajo demanda. </td> <br />  </tr> <br />  <tr> <br />  <td>Creación de flujos de trabajo híbridos </td> <br />  <td>Puede integrar los Servicios multimedia con procesos y herramientas existentes. Por ejemplo, codifique el contenido in situ y, a continuación, cárguelo en los Servicios multimedia para realizar la transcodificación en varios formatos y entregue a través de CDN de Azure o un CDN de otro proveedor. Los Servicios multimedia pueden llamarse individualmente a través de las API de REST estándar para la integración con servicios y aplicaciones externas. </td> <br />  </tr> <br />  <tr> <br />  <td>Ofrecimiento de compatibilidad en la nube para reproductores multimedia </td> <br />  <td>Puede crear, administrar y entregar contenido multimedia en varios dispositivos multimedia (incluidos los dispositivos iOS, Android y Windows) y distintas plataformas. </td> <br />  </tr> <br />  </tbody> <br /> </table> </p>  <h3> <a name="media-client"> </a>Desarrollo de clientes de Servicios multimedia </h3>  <p>Amplíe el alcance de la solución de Servicios multimedia mediante los SDK y los marcos de reproductores para generar aplicaciones clientes multimedia. Estos clientes son para los desarrolladores que deseen generar aplicaciones de Servicios multimedia que ofrezcan experiencias de usuario convincentes en una gama de dispositivos y plataformas. Según los dispositivos de destino, existen opciones para los SDK y marcos de reproductores disponibles en Microsoft y otros socios. </p>  <p>A continuación se muestra una lista de SDK de clientes y marcos de reproductores compatibles. Para obtener más información sobre estos y otros marcos de reproductores y SDK planificados, y la funcionalidad que pueden admitir, consulte  <a href="http://social.msdn.microsoft.com/Forums/es-es/MediaServices/thread/e9092ec6-2dfc-44cb-adce-1dc935309d2a">Desarrollo de clientes de Servicios multimedia </a> en los foros de Servicios multimedia. </p>  <h4 id="mac-and-pc-client-support">Compatibilidad con clientes de PC y Mac </h4>  <p>Para los PC y Mac, puede dirigir una experiencia de transferencia mediante Microsoft Silverlight o Adobe Open Source Media Framework. </p>  <ul>  <li> <a href="http://www.microsoft.com/es-es/download/details.aspx?id=29940">Cliente de transmisión por secuencias suave para Silverlight </a> </li>  <li> <a href="http://smf.codeplex.com/documentation">Microsoft Media Platform: Player Framework para Silverlight </a> </li>  <li> <a href="http://go.microsoft.com/fwlink/?LinkId=275022">Complemento de transmisión suave para OSMF 2.0 </a>. Para obtener información sobre el uso de este complemento, consulte  <a href="../media-services-use-osmf-smooth-streaming-client-plugin/">Uso del complemento Smooth Streaming para Open Source Media Framework de Adobe </a>. </li>  </ul>  <h4 id="windows-8-applications">Aplicaciones de Windows 8 </h4>  <p>Para Windows 8, puede generar aplicaciones de la Tienda Windows mediante una de las construcciones y lenguajes de desarrollo compatibles como HTML, Javascript, XAML, C# y C+. </p>  <ul>  <li> <p> <a href="http://go.microsoft.com/fwlink/?LinkID=246146">SDK de cliente Smooth Streaming para Windows 8 </a>. Para obtener información sobre cómo crear un reproductor de transmisión suave en HTML5, consulte  <a href="http://msdn.microsoft.com/es-es/library/jj573656(v=vs.90).aspx">Tutorial: Generación del primer reproductor de transmisión suave de HTML5 </a>. </p> </li>  <li> <p> <a href="http://playerframework.codeplex.com/wikipage?title=Player%20Framework%20for%20Windows%208%20Metro%20Style%20Apps&referringTitle=Home">Microsoft Media Platform: Player Framework para aplicaciones de la Tienda Windows de Windows 8 </a> </p> </li>  </ul>  <h4 id="xbox">Xbox </h4>  <p>Xbox es compatible con aplicaciones de Xbox LIVE que consumen contenido de transmisión suave. El kit de desarrollo de aplicaciones (ADK) para Xbox LIVE incluye: </p>  <ul>  <li>Cliente de Smooth Streaming para ADK de Xbox LIVE </li>  <li>Microsoft Media Platform: Player Framework para ADK de Xbox LIVE </li>  </ul>  <h4 id="embedded-or-dedicated-devices">Dispositivos dedicados o insertados </h4>  <p>Dispositivos como TV, decodificadores (set-top box), reproductores Blu-Ray, soluciones de TV OTT y dispositivos móviles conectados que tengan un marco de desarrollo de aplicaciones personalizado y un proceso multimedia personalizado. Microsoft proporciona los siguientes kits de migración que pueden ser objeto de licencia, y permite a los socios migrar la reproducción de transmisión suave para la plataforma. </p>  <ul>  <li> <a href="http://www.microsoft.com/es-es/mediaplatform/sspk.aspx">Kit de portabilidad de cliente de Smooth Streaming </a> </li>  <li> <a href="http://www.microsoft.com/PlayReady/Licensing/device _technology.mspx">Kit de portabilidad de dispositivos de Microsoft PlayReady </a> </li>  </ul>  <h4 id="windows-phone">Windows Phone </h4>  <p>Microsoft proporciona un SDK que puede usarse para crear aplicaciones de vídeo premium para Windows Phone. </p>  <ul>  <li> <a href="http://www.microsoft.com/es-es/download/details.aspx?id=29940">Cliente de transmisión por secuencias suave para Silverlight </a> </li>  <li> <a href="http://smf.codeplex.com/documentation">Microsoft Media Platform: Player Framework para Silverlight </a> </li>  </ul>  <h4 id="ios-devices">Dispositivos iOS </h4>  <p>Para los dispositivos iOS, incluidos iPhone, iPod, e iPad, Microsoft incorpora un SDK que puede usar para generar aplicaciones para estas plataformas y ofrecer contenido de vídeo premium: SDK de transmisión suave para dispositivos iOS con PlayReady.
