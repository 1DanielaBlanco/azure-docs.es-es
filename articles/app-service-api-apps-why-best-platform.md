# ¿Qué son las Aplicaciones de API?

## Información general

El Servicio de aplicaciones de Azure es una plataforma de proceso completamente administrada para desarrolladores profesionales que ofrece un amplio conjunto de capacidades para escenarios web, móviles y de integración. Las Aplicaciones de API forman parte del conjunto de aplicaciones del Servicio de aplicaciones y permiten a cualquier usuario técnico o desarrollador descubrir, hospedar, administrar y monetizar conectores de API y SaaS en una plataforma en la nube moderna, repleta de funciones, escalable y globalmente disponible.

## ¿Por qué las Aplicaciones de API?

Aplicaciones de API del Servicio de aplicaciones de Azure facilitan el desarrollo, publicación, administración y monetización de las API.

- **Traiga su API tal como está**: use ASP.NET, Java, PHP, Node.js o Python para las API. Las API pueden sacar partido de las características del Servicio de aplicaciones de Azure sin realizar ningún cambio.

- **Conecte sus aplicaciones a las plataformas de SaaS populares**: el Servicio de aplicaciones de Azure facilita la conexión a plataformas SaaS populares, como Salesforce, Office 365, Twitter, Facebook, Dropbox y muchas más.

- **Control de acceso simple**: las API pueden exponerse solo a las demás aplicaciones del Servicio de aplicaciones de Azure o al público, y el usuario puede agregar autenticación de Azure Active Directory o de servicios de terceros sin realizar ningún cambio en el código.

- **Galería de organización y pública de API**: comparta fácilmente las API con otros equipos de su organización mediante el uso de su propia galería organizativa privada de API. Las API también pueden compartirse públicamente para su consumo por desarrolladores de terceros.

- **Generación automática de SDK**: el Servicio de aplicaciones de Azure puede crear automáticamente el SDK para una variedad de lenguajes, como C#, Java y Javascript, para que sus API esté disponibles para muchas plataformas.

- **Utilice el mejor IDE del mercado**: la integración con [Visual
Studio](/campaigns/visual-studio-2013/) simplifica la creación, depuración, empaquetado y publicación de las API y permite la administración de la aplicación durante todo el ciclo de vida.

- **Acceso a datos locales**: las Aplicaciones de API pueden consumir datos de su propio centro de datos mediante [conexiones híbridas](integration-hybrid-connection-overview.md) y [VNET](web-sites-integrate-with-vnet.md).

- **Implementación sin fricciones**: configure la integración continua y los flujos de trabajo de implementación con Visual Studio Online, GitHub, TeamCity, Hudson o BitBucket, lo que permite compilar, probar e implementar automáticamente la aplicación de API en cada comprobación del código o pruebas de integración correctas.

## Conceptos de Aplicaciones de API ##

- **Galería de Aplicaciones de API**: seleccione entre una lista creciente de plantillas de Aplicaciones de API existentes.
- **Conectores**: un tipo de aplicación de API que facilita la tarea de conectarse a plataformas SaaS como Salesforce y Office 365.
- **Puerta de enlace**: una aplicación web que trata las funciones de administración de API, como la autenticación para todas las Aplicaciones de API en un grupo de recursos. 
- **Escalado automático**: las Aplicaciones de API pueden escalar vertical u horizontalmente de manera automática para controlar cualquier carga entrante de los clientes. Manualmente seleccione el número y tamaño de las máquinas virtuales o configure el escalado automático para escalar los servidores en función de la carga o programación.
- **Integración continua**: configure la integración continua y los flujos de trabajo de implementación con VSO, GitHub, TeamCity, Hudson o BitBucket, lo que permite compilar, probar e implementar automáticamente la aplicación web en cada comprobación del código o pruebas de integración correctas.
