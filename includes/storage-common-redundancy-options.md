Entre las opciones de replicación para una cuenta de almacenamiento se incluyen las siguientes:

* [Almacenamiento con redundancia local (LRS)](../articles/storage/common/storage-redundancy-lrs.md): una estrategia de replicación sencilla y de bajo costo. Los datos se replican en una única unidad de escalado de almacenamiento.
* [Almacenamiento con redundancia de zona (ZRS)](../articles/storage/common/storage-redundancy-zrs.md): replicación para obtener alta disponibilidad y durabilidad. Los datos se replican de forma sincrónica en tres zonas de disponibilidad. 
* [Almacenamiento con redundancia geográfica (GRS)](../articles/storage/common/storage-redundancy-grs.md): replicación interregional para protegerse frente a la falta de disponibilidad de toda una región.
* [Almacenamiento con redundancia geográfica con acceso de lectura (RA-GRS)](../articles/storage/common/storage-redundancy-grs.md#read-access-geo-redundant-storage): replicación interregional con acceso de lectura a la réplica.
